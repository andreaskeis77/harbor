# Harbor Handoff Guide

## Goal

A handoff must allow work to continue in a new chat or a later session without losing the true state.

## Handoff structure

Recommended fields:
- date
- status
- phase
- scope completed
- files added or updated
- validation status
- open points
- next recommended step

## Rule

A handoff is not the roadmap and not the project state.
It is a point-in-time transition record.
