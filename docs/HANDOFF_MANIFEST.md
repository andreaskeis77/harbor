# Harbor Handoff Manifest

## Purpose

This document defines what each Harbor handoff must preserve.

## Every handoff must answer

- What phase are we in?
- What was actually validated?
- What remains only prepared?
- Which files changed?
- What is the exact next recommended step?
- Which gates or checks still matter?

## Canonical files to update with larger transitions

- `docs/MASTERPLAN.md`
- `docs/PROJECT_STATE.md`
- one new `docs/_handoff/HANDOFF_*.md`
